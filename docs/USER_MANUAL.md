# MEMPAS 用户使用手册

> 机电材料查询比价分析系统  
> 面向机电采购业务的报价识别、横向比价、历史价格对照和邀标推荐工具。

---

## 1. 这套系统解决什么问题

MEMPAS 主要用于两个端到端业务场景：

1. **招标比价**  
   上传多家供应商报价文件，系统自动识别供应商、物料、规格、单价等信息，形成横向比价矩阵，并和历史价格基准做偏差判断。

2. **邀标推荐**  
   上传招标文件或材料清单，系统识别本次采购材料，并根据历史合作、供应品类、价格竞争力等信息推荐适合邀请的供应商。

日常用户不需要先理解所有菜单。最常用的是：

- 要比价：进入 **招标比价分析**
- 要推荐邀标名单：进入 **邀标建议**
- 要查历史：进入 **采购数据分析**
- 要维护规则：进入 **系统设置**

## 2. 系统里的数据从哪里来

### 2.1 物料主数据怎么进系统

物料主数据是系统比价的基础。系统里的物料不是只能手工维护，主要有以下来源：

| 来源 | 使用场景 | 系统行为 |
|------|----------|----------|
| 初始历史数据导入 | 系统上线前导入历史采购报价 | 生成已有物料库、供应商库、报价历史 |
| Excel/CSV 批量导入 | 后续补充历史采购价格 | 自动识别物料、供应商、品牌、价格，并写入历史报价 |
| 报价 PDF/图片识别 | 招标比价时上传供应商报价单 | 识别后确认入库，自动创建不存在的物料和供应商 |
| 手工新增标准条目 | 少量补充标准物料 | 在“物料主数据”页面手工新增 |

用户在做比价时，通常不需要先去“物料主数据”里新增物料。上传报价文件后，系统会根据识别结果自动匹配已有物料；如果没有匹配到，会自动创建新的物料条目。

### 2.2 供应商怎么进系统

供应商也可以通过多种方式进入系统：

- 历史数据导入时自动创建。
- 报价文件识别出供应商名称后自动匹配。
- 如果系统中没有该供应商，确认入库时会自动创建。
- 也可以在“供应商管理”页面手工维护。

因此，在“招标比价分析”里，**可以不提前选择供应商**。直接上传多家供应商的报价 PDF，系统会自动识别供应商名称。

### 2.3 历史价格怎么形成

每一条入库报价都会关联到：

- 物料
- 供应商
- 项目
- 品类
- 单价、数量、总价
- 品牌、品牌档位
- 报价日期或批次

这些报价形成历史价格库。系统会基于历史报价计算：

- 合理史低
- 历史均价
- 价格区间
- 偏差率
- 偏差色标

## 3. 使用前需要配置什么

大多数用户可以直接使用默认规则。管理员或业务负责人可以在 **系统设置** 中调整以下规则。

### 3.1 偏差阈值

位置：**系统设置 → 偏差阈值**

偏差阈值用于判断报价是否正常。

| 色标 | 含义 | 默认规则 |
|------|------|----------|
| 正常 | 报价偏差在可接受范围内 | 偏差不超过黄色阈值 |
| 黄色 | 需要关注 | 超过黄色阈值，但未超过红色阈值 |
| 红色 | 异常偏高或偏低，需要重点核查 | 超过红色阈值 |

默认一般为：

- 黄色：5%
- 红色：10%

不同品类可以单独设置。例如桥架、阀门、风机盘管等品类可以有不同阈值。

### 3.2 供应商评分权重

位置：**系统设置 → 评分权重**

供应商评分用于供应商画像、邀标推荐和综合评估。系统按 5 个维度打分：

| 维度 | 说明 |
|------|------|
| 价格竞争力 | 报价相对历史基准的竞争力 |
| 历史合作 | 历史中标、合作次数等 |
| 报价完整度 | 是否完整提交报价 |
| 品牌合规 | 品牌是否已维护档位 |
| 商务条款 | 付款、交期、质保等人工维护信息 |

权重合计必须为 100%。调整后会影响后续供应商评分和推荐排序。

### 3.3 品牌档位

位置：**系统设置 → 品牌档位**

品牌档位用于把品牌按一档、二档、三档进行归类。它主要影响：

- 供应商品牌合规评分
- 报价入库时未知品牌提示
- 后续比价和分析中的品牌参考

如果导入或识别报价时遇到未知品牌，系统会提示补充品牌档位。补充后，该品牌会进入品牌档位表，后续再次出现时自动识别。

## 4. 场景一：招标比价

### 4.1 适用情况

当你已经收到多家供应商报价文件，需要横向比较价格时，使用 **招标比价分析**。

典型文件包括：

- 供应商报价 PDF
- 扫描件图片
- 报价单截图

### 4.2 推荐操作方式：不选供应商，直接上传报价文件

这是当前最推荐的使用方式。

1. 进入 **业务功能 → 招标比价分析**
2. 选择本次比价的 **品类**
3. 项目可选，不选也可以比价
4. **供应商不用选**
5. 点击 **下一步**
6. 上传多家供应商的报价 PDF 或图片
7. 系统自动识别每个文件中的供应商名称和报价明细
8. 用户核对识别结果
9. 点击 **确认入库**
10. 至少确认 2 家供应商后，进入比价结果
11. 查看横向比价矩阵和 AI 分析建议

核心原则：

> 如果报价文件里有供应商名称，就不需要提前手动选择供应商。系统会自动识别并匹配供应商；匹配不到时，确认入库时会自动创建供应商。

### 4.3 什么时候需要手动选择供应商

只有以下情况建议提前选择供应商：

- 报价文件里没有供应商名称
- 扫描件质量差，系统可能识别不出供应商
- 你已经明确知道本轮只比较哪几家供应商
- 你想按供应商逐个上传报价单

选择供应商后，系统会进入“按供应商上传”的传统模式。每个供应商都有一个上传区域。你需要对每家供应商分别：

- 上传报价单并确认入库，或
- 点击使用历史数据跳过上传

### 4.4 报价识别后要核对什么

系统识别完成后，会展示结构化结果。用户应重点核对：

- 供应商名称是否正确
- 物料名称是否正确
- 规格型号是否正确
- 单位是否正确
- 单价是否正确
- 数量是否正确
- 品牌是否正确

如果识别结果有误，可以在确认入库前修改。确认后，系统会把结果写入报价库。

### 4.5 确认入库后系统做了什么

确认入库后，系统会自动完成：

- 标准化物料名称
- 查找已有物料
- 没有匹配到则创建新物料
- 查找已有供应商
- 没有匹配到则创建新供应商
- 写入报价记录
- 计算偏差率和色标
- 记录未知品牌，提示后续维护品牌档位

### 4.6 如何看比价结果

比价结果以横向矩阵展示。

常见列含义：

| 列 | 说明 |
|----|------|
| 材料 | 物料名称和规格 |
| 历史均价 | 历史报价过滤异常值后的均价 |
| 合理史低 | 系统认为可参考的历史低价基准 |
| 供应商报价 | 各供应商本次报价 |
| 偏差率 | 本次报价相对合理史低的偏差 |
| 色标 | 正常、黄色关注、红色异常 |
| 推荐 | 系统按价格和偏差等指标给出的推荐参考 |

### 4.7 合理史低是什么意思

合理史低不是历史最低价的简单最小值。

系统会先过滤明显异常的历史价格，再取过滤后价格中的最低值作为“合理史低”。这样可以避免某些录入错误或极端低价影响比价判断。

偏差率计算方式：

```text
偏差率 = (本次报价 - 合理史低) / 合理史低
```

举例：

- 合理史低为 100 元
- 本次报价为 108 元
- 偏差率为 8%
- 如果黄色阈值是 5%、红色阈值是 10%，则该报价为黄色，需要关注

### 4.8 比价结果如何使用

比价结果适合用于：

- 找出每项材料最低报价
- 判断报价是否明显高于历史水平
- 查看供应商报价完整度
- 形成议价依据
- 输出内部评审材料

注意：系统给出的推荐是辅助决策，不替代人工评审。最终仍需结合供货能力、商务条款、品牌要求、履约风险等因素判断。

## 5. 场景二：邀标推荐

### 5.1 适用情况

当你准备发起招标，但还不确定应该邀请哪些供应商时，使用 **邀标建议**。

典型文件包括：

- 招标文件 PDF
- 材料清单扫描件
- 招标清单图片

### 5.2 操作流程

1. 进入 **业务功能 → 邀标建议**
2. 上传招标文件或材料清单
3. 系统自动识别项目信息和材料清单
4. 用户核对识别出的材料清单
5. 点击 **生成推荐供应商**
6. 系统返回推荐供应商列表
7. 默认勾选排名靠前的供应商
8. 用户可以手动增减勾选
9. 点击 **保存邀请名单**

### 5.3 系统如何推荐供应商

系统会根据招标清单推断涉及品类，再从供应商历史数据中计算推荐结果。

主要参考因素包括：

- 供应商是否做过相关品类
- 历史报价数量和覆盖情况
- 历史价格竞争力
- 合作次数
- 品牌档位命中情况
- 综合评分

推荐结果会展示：

- 供应商名称
- 排名
- 综合评分
- 推荐理由
- 涉及品类

### 5.4 用户需要做什么判断

生成推荐后，用户应重点检查：

- 推荐的供应商是否符合本次招标范围
- 是否需要补充本地熟悉但系统历史较少的供应商
- 是否需要剔除近期履约风险较高的供应商
- 邀请数量是否符合内部流程要求

系统默认只勾选靠前供应商，避免一次性误选过多供应商。用户可以手动勾选或取消。

### 5.5 保存邀请名单后发生什么

点击保存后，系统会生成一条招标记录，并保存本次邀请供应商名单。后续可用于追踪本次邀标过程。

## 6. 物料主数据怎么使用

位置：**数据管理 → 物料主数据**

物料主数据是系统的标准物料库。普通用户不需要每天维护，主要用于查询和少量修正。

### 6.1 页面能做什么

| 功能 | 说明 |
|------|------|
| 查看物料 | 按专业、品类查看标准物料 |
| 搜索物料 | 按名称、编码、规格搜索 |
| 新增标准 | 手工新增物料 |
| 编辑 | 修改名称、规格、材质、单位、推荐品牌等 |
| 属性 | 查看该品类的扩展属性定义 |
| 停用 | 停用不再使用的物料，默认列表不再展示 |

### 6.2 新增物料什么时候用

以下情况才需要手工新增：

- 系统中没有该物料
- 暂时没有报价文件可上传
- 需要先维护标准条目供后续使用

如果你是在做比价，通常直接上传报价文件即可，不必先新增物料。

### 6.3 停用是什么意思

停用不是删除历史数据。

物料停用后：

- 默认物料列表不再显示
- 历史报价仍保留
- 已有比价和历史记录不被破坏

适合停用的情况：

- 物料录入重复
- 标准名称不再使用
- 该物料已被新版标准替代

### 6.4 属性里的“匹配”和“差异”

物料属性用于说明不同品类比价时应该关注哪些字段。

| 角色 | 含义 | 示例 |
|------|------|------|
| 匹配 | 原则上应一致，才适合放在一起比价 | 阀门类型、公称压力、连接方式、泵流量、扬程 |
| 差异 | 可以不同，但需要在评审时说明 | 板材厚度、阀体材质、盘管排数、保温方式 |

当前页面的“属性”主要用于查看各品类的属性定义，帮助用户理解报价是否真正可比。

## 7. 采购数据分析怎么用

位置：**数据管理 → 采购数据分析**

用于查询历史报价。常见用途：

- 查某一类材料历史价格
- 查某家供应商历史报价
- 查某个项目历史采购记录
- 导出历史报价明细

常用筛选条件：

- 品类
- 供应商
- 项目
- 关键字
- 日期范围
- 价格范围

这些历史数据会被用于后续比价中的合理史低、历史均价和偏差判断。

## 8. 常见问题

### Q1：做比价前必须先选供应商吗？

不必须。

推荐方式是：只选择品类，然后直接上传多家供应商报价 PDF。系统会自动识别供应商名称。

只有在报价文件无法识别供应商，或你想固定几家供应商逐个上传时，才需要提前选择供应商。

### Q2：报价 PDF 里识别出来的供应商不存在怎么办？

确认入库时，系统会自动创建该供应商。

### Q3：报价 PDF 里识别出来的物料不存在怎么办？

确认入库时，系统会自动创建该物料，并关联本次报价。

### Q4：为什么有些报价没有偏差率？

通常是因为该物料历史报价不足，系统暂时无法计算合理史低。后续数据积累后会自动改善。

### Q5：为什么有些报价显示红色？

红色表示该报价相对历史基准偏差超过红色阈值。它不是自动判定不能采购，而是提示需要重点核查。

### Q6：配电箱怎么比？

当前系统按同一询价轮次中的配电箱报价进行横向比较。配电箱品牌字段按盘柜制造厂商理解，不按内部元器件品牌拆分比较。

### Q7：邀标推荐能直接代替人工判断吗？

不能。系统推荐是辅助决策，最终仍需业务人员结合项目要求、供应商履约情况、商务条款和内部流程确认。

### Q8：上传识别结果不准怎么办？

在确认入库前，可以人工修改识别结果。确认后才会写入报价库。

## 9. 建议的日常使用方式

### 做一轮比价

```text
招标比价分析
→ 选择品类
→ 不选供应商
→ 上传多家供应商报价 PDF
→ 核对识别结果
→ 确认入库
→ 查看横向比价矩阵
→ 导出或用于评审
```

### 做一次邀标推荐

```text
邀标建议
→ 上传招标文件或材料清单
→ 核对材料清单
→ 生成推荐供应商
→ 调整勾选名单
→ 保存邀请名单
```

### 补充历史价格

```text
采购价格导入
→ 选择品类
→ 上传 Excel/CSV
→ 系统解析
→ 确认导入
→ 自动更新历史价格基准
```

## 10. 用户只需要记住的三句话

1. **比价时，不用先选供应商；直接上传供应商报价 PDF，系统会自动识别。**
2. **物料和供应商会在确认入库时自动匹配或创建，不需要每次手工维护。**
3. **系统给出的是辅助判断，最终采购决策仍需结合业务要求和人工评审。**
